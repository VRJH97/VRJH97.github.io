01
Discovery — The Lab
Experiment generates raw data
A lab sequences tumour DNA from 500 breast cancer patients. The sequencer outputs billions of short nucleotide reads — raw FASTQ files. At this moment, the data exists only on a hard drive in one building. It has no meaning yet.
Illumina Sequencer (instrument)
Lab LIMS system
PI's computer cluster
What happens to the data
→FASTQ reads are aligned to reference genome (GRCh38) using BWA or STAR
→Variant calling identifies mutations vs. reference sequence
→A recurring PTEN nonsense mutation is flagged at Chr10:89,692,905
Key Principle Data only becomes science when it leaves the lab. Most journals now require data deposition as a condition of publication.
↓ MANDATORY DEPOSIT · data submitted before paper is accepted
02
Sequence Archives — Global Infrastructure
Raw data enters the permanent public record
Researchers submit sequences via NCBI's web portal or command-line tools. They fill in metadata: organism, tissue, sequencing platform, experimental conditions. This metadata is what makes data findable. The sequences are immediately mirrored across three continents.
NCBI SRA (raw reads)
GenBank (assembled sequences)
ENA — Europe mirror
DDBJ — Japan mirror
GEO (expression data)
How the INSDC sync works
→NCBI, ENA (EMBL-EBI), and DDBJ sync via INSDC protocol every 24 hours
→Data deposited in any one of the three becomes instantly available from all three
→Each sequence gets a permanent accession number (e.g. GSE123456) — its citation ID forever
→Stored in NCBI data centres in Bethesda, MD — petabytes of sequence data
Infrastructure Reality GenBank stores exabytes of sequence data and serves millions of BLAST queries daily on dedicated compute clusters. It is publicly funded — entirely by NIH.
↓ AUTOMATED PIPELINES · running nightly, no human intervention
03
Annotation — Adding Meaning to Sequence
Databases pull from archives and enrich the data
Ensembl's annotation pipeline pulls fresh sequences from GenBank daily. Computational algorithms identify gene models, regulatory elements, and cross-species conserved regions. Separately, the PTEN protein sequence is translated and pulled into UniProt — first as a computational TrEMBL entry, then enriched by expert curators.
Ensembl (gene annotation)
UniProt TrEMBL → Swiss-Prot
RefSeq (NCBI curated sequences)
InterPro (protein domains)
Pfam (protein families)
Annotation cascade
→GenBank sequence → Ensembl identifies PTEN gene model on Chr10, exon/intron structure
→Protein sequence → UniProt creates entry P60484 (PTEN_HUMAN), adds phosphatase domain via InterPro
→ENCODE adds chromatin accessibility data at the PTEN locus across 200+ cell types
→All annotation cross-referenced: Ensembl gene ID ↔ UniProt accession ↔ HGNC symbol
The Annotation Problem Raw sequence is meaningless without annotation. This step transforms "ATCG…" into "This is the tumour suppressor PTEN, which dephosphorylates PIP3."
↓ SPECIALIST CURATION · expert humans + cross-database linking
04
Variant Classification — Clinical Significance
The mutation enters clinical databases
The specific PTEN variant gets submitted to ClinVar. Curators classify it using evidence from dbSNP (population frequency), OMIM (gene–disease links), and literature. Once classified as Pathogenic, hospital genetic labs worldwide can use this to interpret patient test results.
ClinVar (variant classification)
dbSNP (rsID assignment)
OMIM (gene-disease: #158350)
GWAS Catalog (population risk)
ClinGen (clinical validity)
Clinical data linkage
→Variant → dbSNP assigns rs# (permanent variant ID) after checking it's not already catalogued
→OMIM confirms PTEN mutations cause Cowden Syndrome (OMIM #158350) — loss-of-function mechanism
→ClinVar classifies: Pathogenic (5-star evidence) — this classification propagates to all clinical testing labs
→GeneCards aggregates all this into a single human-readable gene summary page
Translation Moment This is where molecular biology becomes medicine. A ClinVar "Pathogenic" classification directly changes what oncologists tell patients.
↓ PATHWAY CONTEXT · understanding mechanism, not just mutation
05
Drug Discovery — From Target to Therapy
Chemistry and structure databases find the intervention
PTEN suppresses PI3K/AKT/mTOR. With PTEN broken, this pathway is hyperactivated. Drug discovery teams query chemistry databases to find molecules that inhibit downstream targets. Structure databases reveal the 3D binding pocket. Clinical trial registries show what's already being tested.
KEGG (PI3K/mTOR pathway map)
Reactome (reaction-level detail)
PDB (PTEN crystal structure)
AlphaFold (isoform structures)
ChEMBL (AKT inhibitor activity)
DrugBank (approved drugs)
ClinicalTrials.gov
Reaxys (synthesis routes)
From target to candidate
→KEGG/Reactome maps PI3K→AKT→mTOR pathway with every enzyme, substrate, and cross-pathway interaction
→PDB structure of mTOR kinase domain (PDB: 4JSV) reveals binding pocket geometry for drug design
→ChEMBL returns 847 compounds with measured AKT inhibition (IC50 values, ADMET profiles)
→DrugBank confirms rapamycin (sirolimus) is approved — mTOR inhibitor — repurposing candidate
→ClinicalTrials.gov: 23 active trials of mTOR inhibitors in PTEN-deficient cancers currently recruiting
The Chemistry Layer PubChem alone contains 100M+ compounds. ChEMBL has binding data for 2M+ molecules. These databases make drug discovery computational before it becomes experimental.
↓ MULTI-OMICS INTEGRATION · the full molecular picture
06
Integration — The Full Molecular Picture
All data layers converge into systems understanding
The research team integrates genomics, transcriptomics, proteomics, and metabolomics to understand PTEN loss as a whole-system event — not just a single mutation. This is where databases designed for different data types must speak the same language.
TCGA / cBioPortal (cancer omics)
GTEx (normal tissue expression)
Human Cell Atlas (cell types)
ENCODE (chromatin landscape)
STRING (protein interactions)
HMDB / MetaboLights (metabolites)
PhosphoSitePlus (PTMs)
Data integration workflow
→cBioPortal: PTEN mutated in 31% of TCGA breast cancers; co-occurs with PIK3CA mutation; worse survival
→GTEx: normal breast tissue expresses PTEN highly — confirming loss-of-function oncogenic mechanism
→ENCODE: PTEN promoter shows open chromatin in luminal epithelial cells — explains tissue specificity
→STRING: PTEN protein interaction network has 47 direct interactors — each a potential co-target
→HMDB/MetaboLights: PIP3 accumulation (PTEN substrate) seen in tumour metabolomics — validates mechanism in vivo
Why Multi-Omics Matters No single database gives the full picture. The mutation (genomics) → elevated pathway activity (proteomics) → metabolite accumulation (metabolomics) chain only emerges from integration.
↓ PUBLICATION · closing the loop, enriching all databases
07
Publication — Data Feeds Back Into the System
The paper is published. The cycle restarts.
The paper appears in Nature. Within hours, PubMed indexes it. Within days, text-mining algorithms at NCBI and EMBL-EBI scan the full text, automatically updating cross-references in dozens of database records. Human curators are alerted to enrich UniProt, OMIM, and ClinVar entries. Every database touched by this research becomes richer.
PubMed (indexed within hours)
Europe PMC (open access full text)
UniProt (curator flagged)
ClinVar (evidence updated)
Reactome (new pathway step added)
How publication enriches databases
→PubMed ID (PMID) assigned — every database record that cites this paper links to it permanently
→Text-mining identifies: gene names (PTEN, PIK3CA), chemicals (rapamycin), diseases (breast cancer) → automatic cross-links
→UniProt curator reads the paper → adds new functional annotation to PTEN entry P60484
→ClinVar submission referenced in paper → classification elevated from Likely Pathogenic to Pathogenic
↩Other labs read the paper → design new experiments → generate new data → deposit to GenBank → cycle begins again
The Compound Effect Science is cumulative. Each paper enriches 20–50 database records. Those enriched records enable better science by the next team. This is why open access and mandatory data deposition matter — they accelerate the entire cycle.

The Hidden Infrastructure of Science

What makes this system work
🏛️

Public Funding

NCBI runs on NIH funding (hundreds of millions annually). EMBL-EBI is funded by 28 European member states. Without public mandate, none of this would be freely accessible.

🔄

INSDC Agreement

GenBank, ENA, and DDBJ formed the International Nucleotide Sequence Database Collaboration in 1987. All three sync every 24 hours — no sequence is ever lost if one centre fails.

🔗

Accession Numbers

Every sequence, variant, structure, and compound gets a permanent accession number. This is the citation system of data — the equivalent of a DOI for raw science.

🤖

Nightly Pipelines

Databases pull from each other via REST APIs on automated schedules. Ensembl pulls GenBank nightly. UniProt pulls Ensembl weekly. No human orchestrates these — it's all scripted infrastructure.

📐

FAIR Principles

Findable, Accessible, Interoperable, Reusable. The philosophical framework that governs how scientific data infrastructure is designed. Enforced by funders and journals.

🧑‍💻

Human Curation

UniProt Swiss-Prot has ~600,000 expertly curated entries built over decades. OMIM curators read every relevant paper published. Automation finds the data; humans give it meaning.

The Core Insight
Science is not a collection of isolated facts stored in separate databases. It is a single, continuously enriching network of evidence — where every new experiment feeds every prior record, and every prior record enables every new experiment. The databases are not the destination. They are the connective tissue.
